React高阶组件

React 组件间抽象

使用mixin
mixin的特性一直存在于各种面向对象语言中，mixin是为了实现一种类似于多重继承的效果。
先来体验一下用JS封装个mixin方法

对于广义的mixin方法，就是用赋值的方式将mixin对象里面的方法都挂载到对象上，来实现对对象的混入。有些类似于es6里的Object.assign()方法

在React中使用mixin
React在使用createClasss构建组件时提供了mixin属性，比如官方封装的PureRenderMixin

在createClass对象参数中传入数组mixins，里面封装了我们所需要的模块。mixin数组里也可以增加多个mixin。React中不予许有重名的方法，它不会覆盖，而是会报错。
使用createClass实现的mixin为组件做了两件事

  • 共享工具方法
  • 生命周期继承，props与state合并。这是mixins特别重要的功能，它能够合并生命周期方法。如果有很多mixin来定义componentDidMount这个周期，那么React会智能的合并起来执行。同样，mixin也可以作用在getInitialstate的结果上，作state的合并，而props也是这样合并的。

mixin的问题
React社区从0.14版本开始渐渐开始剥离mixin

  • 破坏了组件原有的封装性
  • 命名冲突
  • 增加复杂性

高阶组件
高阶组件类似于高阶函数,它接受React组件作为输入，输出一个新的React组件，通俗的说法就是，当React被包裹的时，高阶组件会返回一个增强的React组件。
实现高阶组件的方法有两种：

  • 属性代理(props proxy)。高阶组件通过被包裹的React组件来操作props。
  • 反向继承(inhertance inversion)。高阶组件继承于被包裹的React组件。
