react通信技巧

react通信技巧整理

从react的嵌套关系上来讲，传递信息有三种可能性

  • 父组件向子组件通信
  • 子组件向父组件通信
  • 没有嵌套关系的组件之间通信

父组件向子组件通信
React数据流动是单向的，父组件通过props向子组件通信，这是React中最常见的方式

子组件向父组件通信

  • 利用回调函数
  • 利用自定义事件机制
    两种方法都可以，回调函数适用于较为简单的情景， 也是最常用的方式，自定义事件机制方法更通用，适用于各种情景，包括之后要说的没有嵌套关系的组件间通信。
  • 回调函数方式

调用Item组件的App组件，在其中添加handleItemChange方法，传递给Item组件，这样在使用List组件时就可以拿到改变的项对应的值

App组件通过handleItemChange回调函数拿到了Item变化的信息，Item组件通过onItemChange拿到了ListItem变化的信息

  • 自定义事件机制
    在没有嵌套关系的组件通信中说

并没有给ListItem传递props，而是在父组件中定义了ChildContext,这样从这一层开始的子组件都可以拿到定义的context，例如这里的color。Context就像一个全局变量，有很强的副作用，官方不建议大量使用context，较好的使用场景是真正意义上的全局且不会更改的信息，例如界面主题，用户信息等。

没有嵌套关系的组件
最合适的通信方式便为自定义事件机制，在处理事件的过程中需要注意，在componentDidMount组件挂载完成，再订阅事件；当组件卸载componentWillUnmount事件中取消事件的订阅。
